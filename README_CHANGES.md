
## Also this round: Profile dropdown

The brief calls for Profile/Preferences/API Keys/Notifications/Logout in the
user menu. Added **API Keys**, **Notifications**, and kept **Change Password**
(the closest real equivalent to "Profile" this app actually has) plus
**Logout** — all linking to real, working destinations. I deliberately did
**not** add stub "Profile" or "Preferences" menu items, since there's no
backend capability behind them yet; adding dead links would violate the
brief's own "no fake data / no dead components" rule. Building real
preferences/profile-editing would be a genuine next increment if you want it.
